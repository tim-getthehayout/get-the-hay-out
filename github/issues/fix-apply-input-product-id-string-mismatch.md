## Context

The "Apply input" dialog (Pastures → Apply Input) cannot save any application that uses an input product. After selecting a product, entering a quantity, and selecting one or more pastures, the user gets the alert:

> "Select a product/source, quantity, and at least one location"

A visible symptom appears earlier: the **Unit** field on the form stays at "—" even after a product is selected, and the NPK info card under the product picker never appears.

### Root cause

Input products use string IDs in the format `IP-XXXXX` (e.g., `IP-00001`). These are generated by `generateInputProductId()` and applied across the data model by `migrateSystemIds()`. The `<select id="ai-product">` correctly renders options with `value="IP-00001"`.

However, three functions in the apply-input flow call `parseInt()` on that value before doing the lookup:

| Line | Function | Code |
|---|---|---|
| 16788 | `onAiProductChange()` | `const id = parseInt(document.getElementById('ai-product').value);` |
| 16862 | `calcAiPreview()` | `const pid = parseInt(document.getElementById('ai-product').value);` |
| 16929 | `saveApplyInput()` | `S.inputProducts.find(x => x.id === parseInt(document.getElementById('ai-product').value))` |

`parseInt("IP-00001")` returns `NaN`. `S.inputProducts.find(x => x.id === NaN)` returns `undefined`. The cascade:

1. `onAiProductChange()` → product not found → unit display stays "—", NPK info card hidden
2. `calcAiPreview()` → product not found → preview hidden, function returns `undefined`
3. `saveApplyInput()` → calls `calcAiPreview()` → gets `undefined` → fires the generic validation alert

### Why manure batches still work

Manure batch IDs are generated with `Date.now()` (numeric). `parseInt()` on a numeric ID string round-trips correctly, so the manure-batch path of the same dialog works fine. Only the product path is broken.

## Requirements

- [ ] Remove `parseInt()` from the `ai-product` value reads in `onAiProductChange()` (~line 16788)
- [ ] Remove `parseInt()` from the `ai-product` value reads in `calcAiPreview()` (~line 16862)
- [ ] Remove `parseInt()` from the `ai-product` value read in `saveApplyInput()` (~line 16929)
- [ ] All three sites should read the value as a string and compare directly against `x.id`
- [ ] Verify the empty-string case (no product selected) still short-circuits cleanly — `S.inputProducts.find(x => x.id === '')` returns `undefined`, which keeps the existing "if (!p) return" guards working
